    Task: Process Control Library (proclib)¶

    In this assignment, you will implement a small process-control library called proclib.

    Inside your repository you will find:

    hw6/
    proclib.h
    proclib.c
    test_proclib.c

    • hw6/proclib.h — contains the function prototypes and documentation.

    • hw6/proclib.c — you must implement the functions in this file.

    • hw6/test_proclib.c — a tester driver that validates your implementation.

    You should not modify proclib.h. You may use the tester to validate your implementation.

    The Command Structure¶

    The library uses the following structure (defined in proclib.h):

    typedef struct {
    char *path;
    char **argv;
} Command;

    • path is the absolute path to the executable (e.g., /bin/ls).

    • argv is a NULL-terminated argument array.

    • argv[0] should contain the program name.

    Example:

    char *args[] = {"/bin/ls", "-l", NULL};
Command cmd = {"/bin/ls", args};

    Problem 1: proclib_run_n_times¶

    Prototype (see proclib.h):

    int proclib_run_n_times(Command cmd, int n);

    Description¶

    Executes the given command n times sequentially.

    For each execution:

    • Call fork()

    • In the child, call execv()

    • In the parent, call waitpid()

    Return Value¶

    Returns the number of successful executions (exit status 0).

    Example¶

    char *args[] = {"/bin/true", NULL};
Command cmd = {"/bin/true", args};

int result = proclib_run_n_times(cmd, 5);

    Expected:

    result == 5

    Problem 2: proclib_run_and_report¶

    Prototype:

    int proclib_run_and_report(Command cmd);

    Description¶

    Executes a command and determines how it terminated.

    You must use:

    • WIFEXITED()

    • WEXITSTATUS()

    • WIFSIGNALED()

    Return Value¶

    • 0 if child exited normally with status 0

    • 1 if child exited normally with nonzero status

    • -1 if child terminated by signal or fork failed

    How to Test¶

    The tester file already includes test cases using:

    /bin/true  // is program that returns an exit status of 0 always.
/bin/false // is a program that returns an exit status of 1 always.

    You can also create a small crashing program and modify the tester to verify signal detection.

    Problem 3: proclib_run_until_failure¶

    Prototype:

    int proclib_run_until_failure(Command cmds[], int n);

    Description¶

    Executes multiple commands sequentially.

    Execution stops immediately when a command fails.

    Return Value¶

    • -1 if all commands succeed

    • Otherwise, return the index (0-based) of the first failing command

    How to Test¶

    The provided tester verifies that:

    • Execution stops at the correct index

    • Later commands are not executed

    You may add additional printf statements if needed.

    Problem 4: proclib_run_parallel¶

    Prototype:

    int proclib_run_parallel(Command cmds[], int n);

    Description¶

    Executes all commands in parallel.

    Important: - Fork all children first. - Then wait for all of them.

    Return Value¶

    Returns the number of commands that exited successfully (status 0).

    How to Test¶

    The tester includes a timing-based test using /bin/sleep.

    If implemented incorrectly (sequentially), runtime will be much longer.

    You may also run manually:

    time ./test_proclib

    Problem 5: proclib_spawn_and_track¶

    Prototype:

    int proclib_spawn_and_track(Command cmd);

    Description¶

    Spawns a child process and prints how it terminated.

    The parent must print either:

    Child PID <pid> exited with status <status>

    or

    Child PID <pid> terminated by signal <signal>

    Return Value¶

    • Exit status if exited normally

    • -1 if terminated by signal or fork failed

    How to Test¶

    The tester already validates:

    • Correct return value

    • Proper reporting behavior

    You should visually verify the printed PID message.

    Compilation Instructions¶

    From inside the hw6 directory:

    gcc proclib.c test_proclib.c -o test_proclib

    Run:

    ./test_proclib

    If all tests pass, you will see a summary message.
